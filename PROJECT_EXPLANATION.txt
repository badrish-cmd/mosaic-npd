1. WHAT IS THIS PROJECT?
================================================================================

Mosaic NPD (New Product Development) is a web application that acts as an
"AI Product Inventor." 

In simple words: You feed it consumer complaints, Reddit discussions, Google
search trends, and competitor product data — and it automatically invents
brand-new product ideas backed by real consumer evidence.

Think of it as a smart assistant for a Product Manager at a company like
Man Matters, Mamaearth, or WOW Skin Science. Instead of spending weeks
reading thousands of reviews and doing market research manually, this tool
does it in seconds and gives you a ready-to-present product concept brief.


================================================================================
2. THE REAL-WORLD PROBLEM IT SOLVES
================================================================================

In the real world, when a company wants to launch a new product, the process
(called NPD — New Product Development) typically looks like this:

   OLD WAY (Manual, takes weeks):
   ───────────────────────────────
   1. Hire a team to read 1000s of Amazon/Flipkart reviews
   2. Manually browse Reddit/forums for consumer pain points
   3. Check Google Trends for what people are searching for
   4. Research competitor products and pricing
   5. Compile all findings into a spreadsheet
   6. Brainstorm product ideas in meetings
   7. Write a product brief document
   8. Present to leadership for approval

   NEW WAY (With this tool, takes minutes):
   ─────────────────────────────────────────
   1. Upload CSV files of reviews, Reddit posts, trends, competitors
   2. Click "Refresh" on the dashboard
   3. Get fully-formed product concepts with evidence, pricing,
      target consumer, ingredients, and a ready-to-present brief

The tool compresses weeks of work into minutes.

4. BACKEND — WHAT HAPPENS BEHIND THE SCENES
================================================================================

The backend is built with Python using the FastAPI framework. Here is what
each file does in simple words:

   app/main.py
   ────────────
   The "front door" of the backend. It starts the server, connects all the
   routes together, sets up the database tables, and allows the frontend
   to talk to it (CORS middleware).

   app/database.py
   ────────────────
   Connects to a PostgreSQL database using SQLAlchemy. Think of it as the
   "bridge" between Python code and the database.

   app/models.py
   ──────────────
   Defines 6 database tables (like Excel spreadsheets in the database):
     1. reviews_raw — stores uploaded customer reviews
     2. reddit_raw — stores uploaded Reddit posts
     3. trends_raw — stores uploaded Google Trends data
     4. competition_products — stores uploaded competitor data
     5. opportunity_clusters — stores calculated opportunity scores
     6. product_concepts — stores the invented product concepts

   app/config.py
   ──────────────
   Reads the database connection URL from environment variables (.env file).

   app/routes/upload.py
   ─────────────────────
   Handles file uploads. When the user uploads a CSV:
     1. Validates that the CSV has the correct columns
     2. Cleans the data (handles missing values, NaN, etc.)
     3. Saves each row to the correct database table
   Supports 5 upload types: reviews, reddit, trends, competition, cumulative.

   app/routes/analysis.py
   ───────────────────────
   The "brain" route. Has 3 endpoints:
     GET /analysis/complaints  → runs complaint mining
     GET /analysis/opportunities → runs opportunity scoring
     GET /analysis/concepts → runs the full pipeline:
       complaint mining → opportunity scoring → concept generation →
       brand fit scoring → save to database → return to frontend

   app/routes/health.py
   ─────────────────────
   A simple "are you alive?" check. Returns status message.

   app/services/review_analyzer.py
   ────────────────────────────────
   The complaint mining engine. Scans every review and Reddit post against
   10 complaint themes using keyword matching. Returns intensity scores
   and actual quotes as citations.

   app/services/scoring.py
   ────────────────────────
   The opportunity scoring engine. Combines complaint intensity + Google
   Trends growth + Reddit buzz - competition pressure to produce a final
   score (0-100) and tier classification for each theme.

   app/services/concept_generator.py
   ───────────────────────────────────
   The product inventor. Takes scored opportunities and generates complete
   product concepts with names, formats, ingredients, pricing, consumer
   profiles, competitive positioning, and 7-section executive briefs.
   Uses built-in knowledge bases:
     - 12 ingredients with benefits
     - 10 product format options with price bands
     - 10 theme-to-concept direction mappings

   app/services/brand_fit.py
   ──────────────────────────
   Scores each concept 0-100 on brand alignment across 4 dimensions:
   gender, price, opportunity strength, and category fit.

   app/services/ai_adapter.py
   ───────────────────────────
   A placeholder for future AI integration (e.g., Claude/GPT API).
   Currently passes text through unchanged. When an API key is provided,
   it could refine executive briefs using large language models.

6. DATABASE — WHERE DATA LIVES
================================================================================

The app uses PostgreSQL (a powerful relational database) with 6 tables:

   INPUT TABLES (raw data from CSV uploads):
   ──────────────────────────────────────────
   reviews_raw          → product_name, brand, category, review_text, rating, source
   reddit_raw           → subreddit, title, post_text, upvotes, keyword_detected
   trends_raw           → keyword, search_volume, growth_percent, timeframe
   competition_products → product_name, brand, category, format, price, ingredient_focus

   OUTPUT TABLES (generated by the analysis engine):
   ──────────────────────────────────────────────────
   opportunity_clusters → theme, complaint_intensity, search_growth, reddit_mentions,
                          competition_density, opportunity_score, tier
   product_concepts     → product_name, persona, format, ingredient_direction,
                          price_range, positioning, supporting_data,
                          opportunity_score, brand_fit_score, tier

   Every table has an auto-incrementing ID and a created_at timestamp.

8. CAPABILITIES
================================================================================

What this tool CAN do:

   a) COMPLAINT MINING
      - Scan thousands of reviews and Reddit posts in seconds
      - Identify 10 distinct consumer pain point categories
      - Calculate intensity (how widespread each complaint is)
      - Provide actual consumer quotes as evidence

   b) TREND ANALYSIS
      - Match consumer complaints with Google search trends
      - Identify themes where demand is growing
      - Calculate search volume and growth percentages

   c) COMPETITIVE LANDSCAPE ANALYSIS
      - Count how many competitors exist in each space
      - Identify format gaps (e.g., everyone makes oils, nobody makes sprays)
      - Calculate competition intensity scores
      - Identify ingredient saturation (what ingredients are overused)

   d) MARKET SIZING
      - Estimate addressable market size from search volume
      - Classify opportunities by market potential
      - Uses Indian market benchmarks (Rs Crore ranges)

   e) PRODUCT INVENTION
      - Generate complete product concepts automatically
      - Suggest creative product names
      - Define target consumer profiles
      - Recommend product formats based on competitive gaps
      - Suggest trending ingredient combinations
      - Provide pricing recommendations
      - Write full executive briefs with 7 sections

   f) BRAND FITNESS EVALUATION
      - Score each concept on brand alignment (0-100)
      - Consider gender targeting, price band, category fit
      - Help filter concepts that match the brand identity

   g) DATA FORMATTING
      - Convert any CSV format to the required column structure
      - Interactive 3-step column mapping wizard
      - Supports all 5 data types

   h) EVIDENCE-BASED OUTPUT
      - Every recommendation is backed by cited consumer evidence
      - Actual review quotes, Reddit post titles, trend data
      - No unsupported claims — everything is traceable

   i) AI CONFIDENCE SCORING
      - Combines opportunity + brand fit into a single confidence metric
      - Helps PMs quickly identify the strongest concept
      - Displayed prominently in the AI Insight Summary

10. CONS & LIMITATIONS
================================================================================

   1. KEYWORD-BASED (NOT TRUE AI/NLP)
      Complaint mining uses simple keyword matching, not machine learning.
      It can miss complaints phrased in unusual ways. For example, "this
      product made my hair worse" wouldn't match any keyword theme.

   2. NO AUTOMATIC DATA COLLECTION
      Users must manually collect and upload CSV data. The tool does not
      scrape Amazon, Reddit, or Google Trends automatically (the scraping
      page is just a "coming soon" placeholder).

   3. NO REAL AUTHENTICATION
      Login is localStorage-based — there is no real backend authentication,
      no password hashing, no user database. Anyone who opens the URL can
      access the dashboard.

   4. SINGLE BRAND PROFILE
      Brand fit scoring is hardcoded for a male-focused, Rs 500-1000,
      haircare/grooming brand. It doesn't adapt to different brand profiles
      without code changes.

   5. NO HISTORICAL TRACKING
      Each analysis run clears previous results. There is no version history
      or trend-over-time tracking of how opportunities change.

   6. LIMITED CATEGORY SCOPE
      The complaint themes, ingredient pool, and format options are designed
      for personal care (haircare, skincare). Using it for unrelated
      categories (electronics, food) would give poor results.

   7. NO EXPORT FUNCTIONALITY
      Product concepts cannot be exported as PDF, PowerPoint, or Excel
      directly from the dashboard. Users must copy-paste.

   8. STATIC INGREDIENT & FORMAT KNOWLEDGE
      The 12 ingredients and 10 formats are hardcoded. New trends
      (e.g., a new miracle ingredient) require code changes.

   9. NO MULTI-USER COLLABORATION
      Single-user tool. No team features, commenting, or shared workspaces.

  10. ENGLISH ONLY
      Cannot process reviews written in Hindi, regional languages, or
      mixed-language (Hinglish) reviews that are common in Indian
      e-commerce.

  11. NO SENTIMENT ANALYSIS
      Treats all keyword matches equally. A review saying "not greasy at
      all" would still trigger the greasy_texture theme because it contains
      the keyword "greasy."

  12. REQUIRES POSTGRESQL
      Database dependency means you need a PostgreSQL server running,
      making local setup slightly complex for non-technical users.

  13. NO AI/LLM INTEGRATION YET
      The ai_adapter.py is a placeholder. The executive briefs are
      template-based, not refined by GPT/Claude. The briefs follow a
      fixed structure and can feel repetitive across concepts.

  14. DATA QUALITY DEPENDENT
      Output is only as good as the input data. Bad or insufficient CSV
      data will produce weak or misleading concepts.

14. FUTURE POSSIBILITIES
================================================================================

Things that could be added to make this tool even more powerful:

   a) REAL AI INTEGRATION
      Connect Claude or GPT API to refine executive briefs, generate
      marketing copy, and produce more creative product names.

   b) AUTOMATIC WEB SCRAPING
      Auto-collect reviews from Amazon, Flipkart, Reddit using web
      scraping. Eliminate manual CSV preparation entirely.

   c) NLP/ML COMPLAINT DETECTION
      Replace keyword matching with machine learning models that
      understand context ("not greasy at all" = positive, not negative).

   d) SENTIMENT ANALYSIS
      Understand not just WHAT consumers say, but HOW they feel.
      Positive, negative, and neutral classification.

   e) MULTI-BRAND PROFILES
      Allow users to configure their own brand profile instead of
      the hardcoded Man Matters-style profile.

   f) PDF/PPTX EXPORT
      One-click export of product concepts as formatted PDF reports
      or PowerPoint presentations.

   g) HISTORICAL DASHBOARD
      Track how opportunities change over time. "Greasy texture complaints
      grew 20% since last quarter."

   h) MULTI-LANGUAGE SUPPORT
      Process Hindi, Hinglish, and regional language reviews that are
      common on Indian e-commerce platforms.

   i) TEAM COLLABORATION
      Multiple users, roles, commenting, and approval workflows.

   j) REAL AUTHENTICATION
      Proper user database, password hashing, JWT tokens, role-based
      access control.

   k) CATEGORY EXPANSION
      Extend beyond personal care to food & beverages, electronics,
      fitness equipment, etc.

   l) A/B TESTING INTEGRATION
      Connect concept scores to actual market test results to validate
      and improve the scoring model over time.
